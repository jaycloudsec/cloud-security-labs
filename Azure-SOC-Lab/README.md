# Azure SOC Lab

## Overview

This repository contains a **hands-on Security Operations Center (SOC) lab environment built in Microsoft Azure**.  
The goal of this project is to simulate real-world blue team workflows including **SIEM deployment, log ingestion, threat detection, incident investigation, and automated response**.

The environment uses **Microsoft Sentinel as the SIEM platform**, with logs collected from Azure virtual machines and other security data sources.

The lab demonstrates practical experience in:

* SOC monitoring
* Incident investigation
* Detection engineering
* Threat visualization
* Security automation

---

## SOC Architecture

Azure resources are used to simulate a security monitoring environment where attacker activity can be detected and investigated.

Main components include:

* Microsoft Sentinel (SIEM)
* Log Analytics Workspace
* Windows Virtual Machines
* Security Event Logs
* Custom Analytics Rules
* Automation Playbooks

Logs generated by simulated attacks are ingested into Sentinel where alerts are triggered and investigated.

## Quick Walkthrough Guide

A simplified step-by-step walkthrough for completing the Azure SOC Lab projects.

For full documentation, screenshots, and investigation details, refer to the individual project folders.

---

## Part 1 — SIEM Deployment & Log Ingestion

1. Sign in to the Azure Portal.

2. Create a Resource Group  
   - Resource Groups → Create  
   - Enter name  
   - Select region  
   - Review + Create → Create

3. Deploy Virtual Machine  
   - Virtual Machines → Create  
   - Select Resource Group  
   - Enter VM name  
   - Select Windows Server image  
   - Configure admin username and password  
   - Enable Public IP  
   - Review + Create → Create

4. Create Log Analytics Workspace  
   - Log Analytics Workspaces → Create  
   - Select Resource Group  
   - Enter workspace name  
   - Select region  
   - Review + Create → Create

5. Enable Microsoft Sentinel  
   - Open Log Analytics Workspace  
   - Microsoft Sentinel → Create  
   - Add workspace

6. Install Azure Monitor Agent  
   - Virtual Machine → Extensions + Applications  
   - Add  
   - Select Azure Monitor Agent  
   - Install

7. Create Data Collection Rule  
   - Monitor → Data Collection Rules → Create  
   - Select Resource Group  
   - Enter rule name

8. Add Data Source  
   - Windows Event Logs  
   - Add **Security**

9. Add Destination  
   - Select Log Analytics Workspace

10. Associate VM  
    - Select deployed VM  
    - Create rule

## Part 2 — Attack Simulation & Threat Detection

1. Open Microsoft Sentinel  
   - Microsoft Sentinel → Select workspace

2. Verify Log Ingestion

```kql
SecurityEvent
| take 10
```

3. Analyze Failed Logins

```kql
SecurityEvent
| where EventID == 4625
| summarize AttemptCount = count() by IpAddress
| sort by AttemptCount desc
```

4. Identify Suspicious IP Addresses  
   - Review query results  
   - Note IPs with highest failed attempts

5. Create Detection Rule  
   - Microsoft Sentinel → Analytics  
   - Create → Scheduled Query Rule

6. Configure Rule  
   - Enter rule name  
   - Set severity  
   - Paste detection query

7. Configure Query Settings  
   - Query frequency: 5 minutes  
   - Lookup period: 5 minutes

8. Configure Alert Logic  
   - Trigger alert when results exceed threshold

9. Review Rule  
   - Next → Review

10. Create Rule  
   - Create

11. Confirm Rule Status  
   - Analytics → Verify rule is enabled
